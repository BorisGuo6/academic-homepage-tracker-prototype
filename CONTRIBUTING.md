# Contributing

Thank you for considering contributing to the Academic Homepage Tracker prototype! Contributions of all kinds are welcome. To help the project run smoothly, please follow these basic guidelines.  

## How to contribute  
- **Fork the repository** and create a new branch for your contribution (e.g. `git checkout -b feature/my-feature`).  
- **Make your changes** in a clear and focused manner. Include tests or examples if applicable.  
- **Commit your work** with a descriptive message (e.g. `git commit -am 'Add feature X'`).  
- **Push to your branch** on your fork (e.g. `git push origin feature/my-feature`).  
- **Open a pull request** against the `main` branch of this repository and provide a summary of your changes and any relevant context.  

## Reporting issues  
If you encounter a bug, performance issue, or have an idea for an enhancement, please [open an issue](../../issues) with as much detail as possible. Include steps to reproduce the problem or a clear description of your suggestion.  

## Code of conduct  
Please be respectful and constructive in your interactions. This project aims to foster a welcoming and inclusive environment for all contributors.  

We appreciate your help in making this project better!
